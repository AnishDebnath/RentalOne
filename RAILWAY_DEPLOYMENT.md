# Railway Deployment Guide - Camera Rental House

Follow these steps to deploy your production-ready architecture.

## 1. Prepare GitHub
- Commit all changes.
- Push your repository to GitHub.

## 2. Create Railway Project
1. Go to [Railway.app](https://railway.app).
2. Click **"New Project"** -> **"Deploy from GitHub repo"**.
3. Select your repository.
4. Railway will automatically detect the `railway.yaml` file and create **three services**: `client`, `admin`, and `api-server`.

## 3. Configuration (Variables)
You need to set the following variables in the **Project Settings** (so they share across services) or in the **api-server** settings:

### Project Variables (Shared)
- `API_URL`: The URL generated by Railway for the `api-server` (e.g., `https://api-server-production.up.railway.app/api`).
- `CLIENT_URL`: The URL generated for the `client` service.
- `ADMIN_URL`: The URL generated for the `admin` service.

### API Server Variables (`api-server`)
Go to `api-server` -> **Variables** -> **New Variable**:
- `SUPABASE_URL`: (From Supabase Dashboard)
- `SUPABASE_ANON_KEY`: (From Supabase Dashboard)
- `SUPABASE_SERVICE_ROLE_KEY`: (From Supabase Dashboard)
- `CLOUDINARY_CLOUD_NAME`: (From Cloudinary)
- `CLOUDINARY_API_KEY`: (From Cloudinary)
- `CLOUDINARY_API_SECRET`: (From Cloudinary)
- `JWT_SECRET`: (Any strong random string)
- `PORT`: `5000`
- `NODE_ENV`: `production`

## 4. Finalizing Domains
1. Once deployed, Railway gives you `xxx.up.railway.app` domains.
2. Update your `API_URL`, `CLIENT_URL`, and `ADMIN_URL` variables to match these if you haven't yet.
3. To add a custom domain later:
   - Go to the service -> **Settings** -> **Domains**.
   - Add `domain.com`, `admin.domain.com`, etc.
   - Update DNS records as instructed by Railway.

## 5. Database Setup
1. In your Supabase project, go to the **SQL Editor**.
2. Run the logic from `migrate_to_single_table.sql` (if you have one) or ensure your schema matches.
3. The app is ready to talk to the DB.

## 6. Troubleshooting
- **Build Fails**: Check logs. If it says "module not found", ensure `NPM_CONFIG_PRODUCTION=false` is set in variables.
- **CORS Error**: Ensure `API_URL` is set correctly in frontends and the origin is allowed in backend.
